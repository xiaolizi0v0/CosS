# CosS v0.5.2 开发记录

## 版本目标

v0.5.1 已经把角色消息、任务指令和 Agent 结构化事件合并为协作时间线，但消息仍停留在 CosS UI 内部。本版本补齐下一步小闭环：允许用户把协作时间线中的消息直接注入到正在运行的目标角色 Agent 终端。

## 已实现内容

### 1. 协作消息注入 Agent 终端

在协作时间线中，如果某条消息的接收角色存在运行中的 Agent 终端，会显示 `注入终端` 按钮。

点击后，CosS 会将消息包装为一段 Agent 指令并写入目标终端：

- 目标角色。
- 发送角色。
- Agent 后端。
- 任务频道。
- 消息来源。
- 原始协作消息内容。
- Agent 输出状态协议提示。

注入时使用 bracketed paste 格式，降低多行内容被 Shell 当成多条命令直接执行的风险。

### 2. 任务指令可同时注入终端

子任务卡片的 `发送给角色` 弹窗新增注入选项：

- 如果目标角色存在运行中的 Agent 终端，默认勾选“同时注入运行中 Agent 终端”。
- 如果没有运行中的目标终端，只保存到协作时间线。

发送后会继续生成 `source=task-instruction` 的任务频道消息，并在可用时同步注入目标角色 Agent 终端。

### 3. 注入状态持久化

消息新增持久化字段：

- `injectedWindowIds`: 已注入的终端窗口 ID。
- `injectedAt`: 最近一次注入时间。

协作时间线会展示 `已注入 N 个终端`，便于用户判断某条消息是否已经投递到 Agent 执行环境。

### 4. 注入日志

新增日志事件：

- `agent.instruction.injected`: 成功注入 Agent 终端。
- `agent.instruction.inject.skipped`: 没有找到运行中的目标 Agent 终端。

日志包含项目、消息、任务、来源角色、目标角色、终端窗口 ID 和内容长度。

## 本轮测试范围

本版本只跑 v0.5.2 相关模块测试，不跑完整 e2e。

计划测试：

- 启动基础壳版本号。
- 标题栏和应用信息版本号。
- 消息中心兼容手动消息。
- 子任务指令写入协作时间线。
- 协作时间线消息注入运行中 Agent 终端。
- Codex 假终端结构化事件同步仍可用。

## 暂未实现内容

以下功能仍保留到后续版本：

- Claude Code / Codex 登录态真实远程 API 校验。
- 对正在运行的 Agent 终端做可确认、可取消、可重试的投递队列。
- 将注入消息和终端输出建立可点击引用关系。
- 任务计划可视化编辑，例如增删子任务、切换角色、修改描述。
- 文件编辑器文件树、新建文件夹、重命名、删除、另存为。
- 浏览器多标签页、收藏/历史、测试角色一键打开任务 URL。
