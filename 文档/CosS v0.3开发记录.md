# CosS v0.3 开发记录

主题：系统模型配置与可选模型切换

## 1. 版本目标

v0.3 开始引入模型配置层，为后续真实 LLM Gateway、任务拆解和角色协作编排做准备。

本阶段先完成：

- 应用版本升级为 `v0.3`。
- 设置页新增模型配置区域。
- 默认系统模型固定为内网模型，无需 API key。
- 可选模型需要用户填写 API key 后才能切换。
- 新建任务会记录当前使用的模型配置。

## 2. 默认系统模型

默认模型配置：

```env
LLM_BASE_URL=http://10.21.1.61:26962/v1
LLM_MODEL_NAME=agent-brain
```

默认系统模型不需要 API key，作为 CosS 的兜底模型配置。

## 3. 可选模型

设置页支持以下可选模型：

- DeepSeek API
- GLM
- OpenAI
- Claude Code

这些模型都需要用户自行填写 API key。未填写 API key 时，只允许编辑配置，不允许切换为当前模型。

## 4. 当前行为

- 点击模型卡片会进入该模型的配置编辑状态。
- 系统默认模型的 Base URL 和模型名为只读。
- 可选模型可编辑 Base URL、模型名称和 API key。
- 点击“设为当前模型”时，如果模型需要 API key 且未填写，系统会保持当前模型不变。
- 填写 API key 后可切换到对应模型。
- 新建任务弹窗会显示当前模型。
- 创建任务时会把模型供应商、Base URL、模型名称和是否有 API key 记录到任务对象中。
