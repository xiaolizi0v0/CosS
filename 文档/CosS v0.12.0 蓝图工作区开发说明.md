# CosS Desktop v0.12.0 蓝图工作区开发说明

## 版本目标

v0.12.0 开始实现 CosS 第三大工作区：蓝图工作区。蓝图与项目、Agent 世界并列，面向同一个目标——完成用户任务并形成可交付结果；不同点是蓝图通过可视化节点和连线表达执行过程。

## 本次已实现

- 侧边栏新增“蓝图”一级入口、蓝图列表、创建、切换和删除流程。
- 蓝图定义随应用状态保存与恢复，旧状态会自动补全蓝图字段。
- 参考“角色消息蓝图”建立网格画布、贝塞尔连接、端口圆点、彩色节点顶边与自动分层布局。
- 建立节点库、流程画布、属性检查器三栏编辑布局。
- 首批提供 32 类节点，覆盖生命周期、AI 协作、工具动作、数据处理、流程控制、人工协作和可靠性。
- 支持添加节点、拖动节点、选择节点/连接、创建连接、删除所选、分类与搜索节点。
- 画布支持 50%–200% 缩放、重置、适应视图、按钮/Space/中键平移、Ctrl/Shift 多选、拖框选择和多节点整体拖动。
- 支持复制、粘贴、重复、撤销、重做与 Delete、Ctrl+A/C/V/D/Z/Y 等画布快捷键；拖动节点时连接线实时跟随。
- 节点卡片支持多个带类型的输入/输出引脚，区分执行流与文本、对象、数组、数字等数据流；数据连接只提供节点输入，不参与流程调度。
- 支持从输出引脚拖拽到兼容输入引脚完成连线，拖动时显示贝塞尔预览、候选端口高亮，并阻止执行流与数据流误连。
- 支持左/右/顶/底对齐、横向/纵向分布、24px 网格吸附、可点击迷你地图以及节点/画布/分组右键菜单。
- 支持把多选节点建立可折叠分组；折叠后保留分组卡片和跨分组连接，内部节点与内部连线暂时隐藏。
- 支持编辑节点名称、说明、执行指令、启用状态、超时和重试次数，以及连接标签。
- 新蓝图默认生成“任务开始 → 规划器 → Agent 任务 → 任务完成”基础流程。
- 支持结构检查：唯一开始节点、完成节点、节点可达性、断头节点和 Agent 指令缺失。
- 支持根据当前蓝图创建任务记录，保存目标、工作目录、蓝图版本和就绪状态。
- 新增持久化图执行器：冻结任务蓝图版本，维护执行队列、节点状态、上下文、重试、日志与交付物。
- 支持任务启动、暂停、继续、取消、失败节点重试以及应用重启后的安全恢复。
- 支持条件与多路路由、并行调度、汇合等待、分支跳过、等待定时器和错误捕获路径。
- 任务开始、完成、失败、变量、模板、解析、拆分、合并、Schema 校验、交付物等节点可直接执行。
- Planner 接入现有模型规划服务，Agent 节点接入统一的本地 Agent 执行通道。
- Agent 节点支持继承全局设置，或独立选择 Claude、Codex、CodeBuddy、模型与权限模式。
- For Each 支持选择逐项执行节点、当前项变量、1–12 并发、失败即停或继续策略，并保存每项状态、重试、输出和错误。
- 支持从任意已完成、失败或跳过节点开始重跑其下游流程，以及删除不再需要的运行历史。
- MCP 工具节点支持读取任务目录 `.mcp.json`、启动 stdio Server、完成 MCP 初始化握手并调用指定工具。
- MCP 工具节点同时支持 Streamable HTTP：JSON/SSE 响应、协议版本、Session ID、工具发现及会话释放。
- 浏览器节点支持打开外部地址、后台加载、整页/选择器提取，以及受审批的等待、点击、填写和提取动作序列。
- For Each 支持分别选择子图开始与结束节点，让每个集合项沿确定路径执行多个节点并记录逐步输出。
- 文件、命令和浏览器节点接入真实本地能力；命令、写文件、删除文件和 MCP 调用执行前必须由用户批准。
- 新增运行面板与画布状态：显示运行中、等待、完成、失败、跳过、节点输出、事件日志和交付物。
- 应用、Kernel、状态契约与 MCP 服务版本统一升级到 `0.12.0`。

## 当前执行边界

本次版本已经完成蓝图编辑器与图执行器的第一阶段。确定性节点、Planner、CodeBuddy Agent、文件、命令和浏览器适配器能够产生真实节点状态与输出；人工节点和副作用操作会进入持久化等待状态。

当前仍有以下边界：MCP 支持 stdio 与 Streamable HTTP，但尚未兼容旧版 HTTP+SSE 双端点；浏览器动作支持等待、点击、填写和提取，但不自动下载文件；For Each 多节点子图要求每个实例在任一节点只选择一条确定路径，暂不允许在实例内部并行、人工等待或嵌套 For Each。界面不会把这些边界伪装成已经自动完成。

## 数据结构摘要

- `state.blueprints`：蓝图集合。
- `state.activeBlueprintId`：当前蓝图。
- `blueprint.nodes`：节点定义、坐标、运行策略以及由节点类型决定的多引脚布局。
- `blueprint.edges`：有向连接、源/目标引脚、执行/数据类型及标签。
- `blueprint.groups`：节点分组、颜色、成员与折叠状态。
- `blueprint.tasks`：基于该蓝图创建的任务记录。
- `blueprint.ui`：单选/多选、待连线、分类、缩放、平移模式与画布滚动状态。

## 后续开发顺序

1. 旧版 HTTP+SSE MCP 兼容与 OAuth 授权流程。
2. 浏览器下载产物登记、动作截图和失败现场保存。
3. For Each 子图内部并行、人工等待与嵌套逐项运行。
4. 运行历史对比与节点输出差异查看。
5. 覆盖矩阵、质量门报告和最终任务完成判定增强。

MCP HTTP 传输实现遵循官方 Streamable HTTP 要求，包括每条消息单独 POST、同时接受 `application/json` 与 `text/event-stream`、协商协议版本以及可选 `Mcp-Session-Id`。参考：[MCP Transport Specification](https://modelcontextprotocol.io/specification/2025-06-18/basic/transports)。
