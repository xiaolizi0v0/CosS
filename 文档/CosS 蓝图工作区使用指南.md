# CosS 蓝图工作区使用指南

蓝图工作区用于把一次用户任务编排成可重复执行的流程。你可以先让系统根据任务目标生成初稿，再通过节点、引脚和连接进行微调，最后使用同一张蓝图创建任务并查看运行状态。

## 1. 打开蓝图工作区

1. 在左侧导航选择“蓝图”。
2. 点击“新建蓝图”创建空白流程，或点击“根据任务生成蓝图”让规则引擎生成初稿。
3. 生成蓝图不会覆盖当前蓝图；生成后仍可在画布中移动、连线、分组和修改属性。

蓝图会保存在 CosS 的本地状态中。默认工作目录只是运行时的建议路径，创建任务时仍可以重新选择。

## 2. 创建蓝图的两种方式

### 手动创建

“新建蓝图”会提供以下默认节点：任务开始、规划器、Agent 任务、任务完成。填写蓝图名称、用途说明和默认工作目录后创建即可。

### 根据任务生成

在生成窗口填写任务目标，可选填输入文件、交付文件和工作目录，并选择生成方式：

- 生成后确认：生成蓝图并打开画布，适合先检查结构。
- 仅生成草稿：只生成蓝图，不创建任务。
- 生成并创建任务：生成蓝图后立即创建一次运行记录。

系统会根据目标识别文件、命令、网页、审阅、交付和审批需求，自动选择节点并填写基础属性。自动生成的 Agent 指令只描述节点动作；本次运行的整体目标在“使用蓝图创建任务”时填写。

## 3. 节点与属性

左侧“节点库”按类别提供 32 种节点。点击节点右侧的“＋”即可添加到画布；选中节点后，在右侧属性检查器中编辑名称、说明、专属配置、超时、重试次数和启用状态。

节点属性的输入规则：

- 执行指令：仅 Agent 任务、上下文镜头、知识检索、评价器和汇总器显示，用来描述该节点的具体动作。
- 评价指令：评价器使用，用来说明评分标准和通过条件。
- 模板内容：模板、命令、文件写入、MCP 参数等节点使用模板字段；支持运行时变量替换。
- 不需要指令或模板的节点不会显示这两个编辑器，避免误以为属性会生效。
- 超时为 0 时使用节点默认值；Agent 节点默认执行时限为 15 分钟。重试次数只影响可重试的执行节点。

常用节点类别：

| 类别 | 典型节点 | 用途 |
| --- | --- | --- |
| 生命周期 | 任务开始、任务完成、任务失败 | 定义任务入口和出口 |
| AI 与协作 | 规划器、Agent 任务、知识检索、评价器 | 拆解目标、执行、检索和评估 |
| 工具与动作 | 文件、命令、浏览器、MCP 工具、交付物 | 访问工作区或外部能力 |
| 数据处理 | 变量、模板、变换、解析与提取、合并 | 组织和转换数据 |
| 流程控制 | 条件、多路分支、并行、汇合、逐项执行、等待 | 控制路径和并发 |
| 人工协作 | 人工输入、人工审批、人工审阅与编辑、交付质量门 | 在流程中暂停等待用户 |
| 可靠性 | 错误捕获 | 进入补救或降级路径 |

## 4. 如何连线

蓝图连接分为流程连接和数据连接。引脚位于节点卡片内部：左侧是输入，右侧是输出；实心引脚表示已连接，空心引脚表示尚未连接。

1. 按住节点右侧的输出引脚。
2. 拖动到目标节点左侧的输入引脚。
3. 松开鼠标完成连接。连接过程中预览线的终点会跟随鼠标指针。
4. 连接必须兼容：流程引脚只能连接流程引脚；数据引脚需为相同类型或 `any`。
5. 点击连线可在右侧编辑标签；按 Delete 或右键菜单可删除连接。

如果只想快速连接默认流程，也可以先点击输出引脚，再点击目标输入引脚。工具栏的“取消连线”会清除当前待连接状态。

## 5. 画布操作

- 拖动节点：按住节点标题区域移动；开启“吸附”后会对齐到 24px 网格。
- 多选：按住 Ctrl/Cmd 或 Shift 点击节点。
- 对齐/分布：使用工具栏或右侧多选面板中的左对齐、顶对齐、横向分布和纵向分布。
- 分组：多选节点后点击“分组”；点击分组标题可以折叠/展开，右键菜单可取消分组。
- 复制/粘贴/重复：支持 Ctrl+C、Ctrl+V、Ctrl+D，也可使用工具栏按钮。
- 缩放/平移：使用缩放按钮或滚轮；按住 Space 拖动画布。点击右下角迷你地图可以快速定位。
- 自动整理：按流程依赖重新排列节点，不会删除节点或连接。
- 撤销/重做：支持 Ctrl+Z、Ctrl+Y。

## 6. 创建并运行任务

1. 点击“使用蓝图创建任务”。
2. 填写“本次运行目标”。这是规划器和 Agent 的统一上下文，节点执行指令只描述具体动作。
3. 选择工作目录，确认蓝图结构检查通过后点击“创建任务”。
4. 在任务记录中点击“查看运行”，可看到节点状态、执行动画、事件日志和交付物。
5. 运行中的命令、文件写入、网页操作和 MCP 副作用会先进入等待审批状态；在运行面板中批准、拒绝或填写人工输入后继续。

运行记录会冻结创建任务时的蓝图版本。之后修改蓝图不会改变已经创建的任务。

## 7. 运行状态与故障处理

节点状态包括：未运行、运行中、等待中、已完成、失败、已暂停、已跳过和已取消。运行面板顶部汇总已完成节点、失败节点和交付物数量；节点卡片会显示当前执行状态动画。

- 失败节点：在运行面板点击“重跑”，只重新执行该节点及其后续可达路径。
- 暂停任务：点击“继续运行”恢复等待中的任务。
- 结构错误：点击“检查蓝图”，按问题列表定位到对应节点。
- MCP 工具：在 MCP 工具节点属性中填写 Server 名称后点击“发现 Server 工具”。
- 删除记录：删除只移除 CosS 中的冻结蓝图、日志和交付物记录，不会删除工作目录中的文件。

## 8. 任务目标与执行指令的边界

建议按以下方式填写：

- 任务目标：写最终效果，例如“读取 package.json，分析项目并生成中文报告”。
- 规划器：写拆解策略或步骤上限。
- Agent 执行指令：写当前节点要完成的动作，例如“读取上游计划，分析依赖并输出风险清单”。
- 文件/命令/模板节点：填写路径、命令或模板；这些节点不使用 Agent 执行指令字段。

这样可以避免每次创建任务时把同一个目标重复写入多个节点，同时保留用户在蓝图中的微调能力。

## 9. 多语言

在“设置 → 系统 → 语言”中切换界面语言。蓝图工作区、节点库、属性检查器、运行面板和 Agent 世界地图会立即重新渲染；当前版本完整提供简体中文和英文文本，其他语言会使用英文蓝图文案作为兼容回退。

## 10. 推荐的最小测试蓝图

```
任务开始 → 规划器 → Agent 任务 → 文件（写入） → 任务完成
```

将“任务开始”的 `目标` 数据引脚连接到“规划器”的 `目标` 输入，将“规划器”的 `计划` 连接到 Agent 的 `上下文`，再把 Agent 的 `结果` 连接到文件节点的 `内容`。最后连接所有流程引脚。运行前确认文件节点的路径模板和写入内容模板，并在审批卡片中批准写入。
