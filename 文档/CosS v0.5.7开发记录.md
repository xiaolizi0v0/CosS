# CosS v0.5.7 开发记录

开发日期：2026-06-29

## 版本目标

v0.5.7 聚焦“对话内任务管理增强”。在 v0.5.5 已经完成“项目 -> 对话 -> 任务”的基础上，把任务列表窗口从单纯展示升级为可检索、可筛选、可查看详情、可归档恢复的任务管理入口，方便用户在一个对话桌面中持续发布多个任务后仍能快速定位上下文。

## 已完成模块

1. 版本升级
   - `package.json` / `package-lock.json` 升级到 `0.5.7`。
   - 应用侧边栏、关于信息、测试断言升级到 `v0.5.7`。
   - 协作时间线标题升级为 `v0.5.7 协作时间线`。

2. 任务列表筛选
   - 新增任务搜索框，支持按任务标题、目标、规划摘要、模型、角色、子任务标题和子任务描述搜索。
   - 新增角色筛选。
   - 新增状态筛选。
   - 新增模型筛选。
   - 新增“显示归档”开关，默认隐藏已归档任务。

3. 任务详情面板
   - 任务列表改为左侧任务列表、右侧详情面板。
   - 详情面板展示任务目标、当前状态、子任务完成数、消息数、投递数、输出引用数和模型名称。
   - 展示子任务角色、标题、说明和状态。
   - 展示关联投递记录，包含目标角色、投递状态和投递方式。
   - 展示最近任务消息，帮助用户快速恢复上下文。

4. 任务归档与恢复
   - 任务支持 `archived` / `archivedAt` 字段。
   - 任务详情中可归档当前任务。
   - 开启“显示归档”后可恢复已归档任务。
   - 归档和恢复都会写入应用日志：
     - `task.archived`
     - `task.restored`

5. 任务列表窗口优化
   - `task-list` 程序默认窗口尺寸从窄展示升级为更适合双栏布局的宽窗口。
   - 保留重复打开聚焦已有任务列表窗口的行为，不重复创建。
   - 筛选输入框统一使用当前应用的标准输入框风格。

## 定向测试范围

本版本不需要跑完整 e2e，建议只跑当前模块相关测试：

```powershell
node --check src\main.cjs
node --check src\renderer.js
node --check src\preload.cjs
node --check tests\e2e\app.spec.cjs
npm.cmd run test:e2e -- tests/e2e/app.spec.cjs -g "v0.5.7|renders v0.5.7|boots into the workspace shell"
```

新增 e2e 覆盖：

- 打开当前对话任务列表。
- 搜索任务。
- 按角色、状态、模型筛选任务。
- 默认隐藏归档任务。
- 查看任务详情中的消息、投递方式和模型信息。
- 归档任务并验证持久化状态。
- 显示归档任务并恢复。
- 验证 `task.archived` / `task.restored` 日志。

## 后续建议

1. 后续可继续增强任务列表排序能力，例如按创建时间、更新时间、状态优先级排序。
2. 后续可增加任务批量操作，例如批量归档、批量恢复、按对话导出任务报告。
3. 后续可把任务详情中的投递记录进一步打通到对应 Agent 终端输出片段，形成更完整的任务追踪链路。
